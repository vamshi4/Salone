# Salone Design Direction

Salone is a product app, not a marketing site. Screens should feel professional, calm, and operational.

## UI Rules

- Prefer clear task flows over decorative layouts.
- Keep cards for real repeated items or forms, not every section.
- Use compact spacing and readable hierarchy.
- Use icons for actions where familiar.
- Avoid purple-only palettes and generic AI gradients.
- Avoid oversized hero sections inside the app.
- Ensure buttons and text never overflow on mobile.
- Keep booking actions obvious: Confirm, Reject, Reschedule, Request.
- Empty/error states should be actionable with Retry or next step.

## Current Visual Lane

- Customer app: polished consumer marketplace, friendly but practical.
- Stylist app: compact dashboard for repeated daily work.
- Salon admin app: operational dashboard, scan-first, action-first.

## UI Polish Order

1. Salon admin booking list/details.
2. Customer booking/reschedule screens.
3. Stylist dashboard settings and availability.
4. Customer discovery cards.
